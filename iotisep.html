Now here's the story...

Physical Architecture:

3 sensors A, B and C measure the humidity of their locations.
We have these sensors and locations pre-registered on our platform. The measured data is processed and sent to a server via wifi using microcontroller.
This dashboard then displays these measured data as well as computes the status of the sensor based on our monitoring algorithm:
  • Far below a pre-set threshold value, status is ok.
  • Close to the threshold, status is leakage eminent.
  • Equal and above the threshold value, leakage is detected.
This dashboard also fields a map of the locations of our sensors.
  • Red indicates leakage detected status.
  • Yellow indicates leakage eminent status.
  • Green indicates OK status.
The application works in real time. Data is refreshed every 5 minutes

Challenges faced

Initial objective was to demonstrate using a long range IoT network like LoRa.
However, due to budget challenges, we have adjusted to make this demonstration using wifi.
This is although functional, it is not practical to depend on or use wifi connectivity in the field.
Dashboard application is static to 3 sensors already pre-set.

Recommendations

We recommend further refinement on this project.
A few points for improvement:

  1. Implement on Long Range, Low Power Wide Area Network.
  2. Dashboard to be made interactive so we can add more sensors and retrieve their data too
